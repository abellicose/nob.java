Format:

```
<type>: <short summary>

<optional body>
```

Types:
- `feat` — new feature
- `fix` — bug fix
- `refactor` — restructuring, no behavior change
- `chore` — build, config, tooling
- `docs` — documentation

Summary is imperative, present tense, no period. "Add target class" not "Added target class".

Body only when the why isn't obvious.
